Backend API contract
====================

Base URL:
http://localhost:8000

All normal API responses, except GET /health, have this wrapper:
{
  "ok": true,
  "data": {}
}

404 errors use FastAPI shape:
{
  "detail": "USER_PROFILE not found"
}

=======================

Endpoints
=========

1. GET /health
---------------
Logic:
- Checks that backend process is alive.
- Does not touch database.

Response:
{
  "status": "ok"
}


2. POST /api/profile/analyze
----------------------------
Logic:
- Creates or updates UserProfile by telegram_id.
- Runs local trained classifier first: bootstrapped Naive Bayes + evidence gates.
- Then tries Gemini/OpenAI-compatible LLM for semantic enrichment.
- If LLM fails, returns 200 with classifier fallback.
- Saves extracted fields into USER_PROFILE.

Body:
{
  "telegram_id": 123,
  "username": "user|null",
  "first_name": "User|null",
  "last_name": "User|null",
  "user_message": "Я хочу стать Python backend разработчиком, знаю основы, 5 часов в неделю, люблю практику",
  "dialog_history": []
}

Response data keys:
{
  "classifier_output": {
    "User_profile_update": {
      "Goal_text": "string|null",
      "Direction": "string|null",
      "Specific_track": "string|null",
      "Target_role": "string|null",
      "Current_level": "beginner|basic|professional|null",
      "Time_per_week_value": 5,
      "Time_per_week_label": "string|null",
      "Preferred_formats": ["practice", "article"],
      "Preference_json": {}
    },
    "signals": {
      "goal": [],
      "level": "string|null",
      "time": "string|null",
      "formats": []
    }
  },
  "llm_output": {
    "Db_target": "USER_PROFILE",
    "Action": "ask_question|ready_for_roadmap",
    "Understood_request": "string",
    "Extracted_facts": {},
    "User_profile_update": {},
    "Missing_fields": [],
    "Assumptions": [],
    "Need_question": false,
    "Next_question": {
      "Type": "goal|specificity|level|time|format|preferences|blocked_source_detail|none",
      "Text": "string",
      "Buttons": [],
      "Allow_multiple": false
    },
    "Ready_for_roadmap_generation": true,
    "Classifier_output": {},
    "Backend_decision": {}
  },
  "fallback_output": "same shape as llm_output|null, present only when LLM failed",
  "llm_status": "failed_classifier_fallback, present only when LLM failed",
  "user_profile": "UserProfile"
}

Important:
- Normal success may not include "fallback_output" and "llm_status".
- If fallback is used, "llm_output" is null.


3. GET /api/profile/{telegram_id}
---------------------------------
Logic:
- Reads USER_PROFILE by telegram_id.
- Returns 404 if profile does not exist.

Response data:
{
  "profile": "UserProfile",
  "user_profile": "UserProfile"
}

"profile" and "user_profile" are the same object. "user_profile" is kept for old frontend compatibility.


4. PATCH /api/profile/{telegram_id}
-----------------------------------
Logic:
- Reads USER_PROFILE by telegram_id.
- Updates only fields sent in body.
- Returns updated profile.

Allowed body fields:
{
  "username": "string|null",
  "first_name": "string|null",
  "last_name": "string|null",
  "goal_text": "string|null",
  "direction": "string|null",
  "specific_track": "string|null",
  "target_role": "string|null",
  "goal_reason": "string|null",
  "current_level": "beginner|basic|professional|null",
  "time_per_week_label": "string|null",
  "time_per_week_value": 5,
  "preferred_formats": ["video", "practice", "article"],
  "wishes": "string|null",
  "preference_json": {},
  "notification_settings_json": {},
  "dialog_state": "string|null",
  "profile_json": {}
}

Response data:
{
  "user_profile": "UserProfile"
}


5. GET /api/profile/{telegram_id}/state
---------------------------------------
Logic:
- Reads profile.
- Reads active/current roadmap.
- If roadmap exists, returns roadmap, items and progress.
- If roadmap does not exist, roadmap/progress are null and items are [].

Response data:
{
  "profile": "UserProfile",
  "roadmap": "Roadmap|null",
  "items": ["RoadmapItem"],
  "progress": "Progress|null",
  "user_profile": "UserProfile",
  "current_roadmap": "Roadmap|null",
  "roadmap_items": ["RoadmapItem"]
}


6. GET /api/profile/{telegram_id}/roadmaps
------------------------------------------
Logic:
- Reads profile.
- Returns user's roadmaps ordered by updated_at desc.

Query:
?status=active
?limit=20

Allowed status:
draft, active, paused, completed, replaced, archived

Response data:
{
  "profile": "UserProfile",
  "roadmaps": ["Roadmap"],
  "count": 0,
  "status_filter": "active|null"
}


7. GET /api/profile/{telegram_id}/roadmap/current
-------------------------------------------------
Logic:
- Reads profile.
- Finds current roadmap: active first, then latest not archived.
- Returns 404 if no roadmap exists.

Response data:
{
  "roadmap": "Roadmap",
  "items": ["RoadmapItem"],
  "progress": "Progress",
  "roadmap_items": ["RoadmapItem"]
}


8. GET /api/profile/{telegram_id}/progress
------------------------------------------
Logic:
- Reads profile.
- Reads current roadmap if it exists.
- Returns progress for current roadmap.

Response data:
{
  "profile": "UserProfile",
  "roadmap": "Roadmap|null",
  "progress": "Progress|null",
  "user_profile": "UserProfile",
  "current_roadmap": "Roadmap|null"
}


9. POST /api/roadmap/generate
-----------------------------
Logic:
- Reads saved UserProfile.
- Tries LLM roadmap generation first.
- If LLM fails or is rate-limited, creates classifier template roadmap.
- Saves Roadmap, RoadmapItems and MotivationPushes.
- Updates UserProfile dialog_state.

Body:
{
  "telegram_id": 123,
  "dialog_history": [],
  "current_datetime": "ISO datetime|null"
}

Response data:
{
  "generation_status": "ok|llm_failed_template_fallback",
  "llm_output": {
    "User_profile_update": {},
    "Roadmap_insert": {},
    "Roadmap_items_insert": [],
    "Motivation_pushes_insert": []
  },
  "user_profile": "UserProfile",
  "created": {
    "roadmap": "Roadmap",
    "items": ["RoadmapItem"],
    "pushes": ["MotivationPush"]
  }
}


10. GET /api/roadmap/{roadmap_id}
---------------------------------
Logic:
- Reads roadmap by roadmap_id.
- Reads all items.
- Reads progress.
- Reads feedback history.

Response data:
{
  "roadmap": "Roadmap",
  "items": ["RoadmapItem"],
  "progress": "Progress",
  "feedback": ["RoadmapFeedback"],
  "roadmap_items": ["RoadmapItem"],
  "roadmap_feedback": ["RoadmapFeedback"]
}

15. POST /api/roadmap/item/{item_id}/start
------------------------------------------
Logic:
- Checks profile by telegram_id.
- If item belongs to profile and is not_started, marks it in_progress.
- If it is already started/completed, returns existing item.

Body:
{
  "telegram_id": 123
}

Response data:
{
  "roadmap_item": "RoadmapItem"
}

17. POST /api/roadmap/item/complete
-----------------------------------
Logic:
- Checks profile by telegram_id.
- Completes item or moves it to pending_check.
- Calculates pending_xp.
- Updates profile XP/streak if repository logic confirms completion.

Body:
{
  "telegram_id": 123,
  "item_id": "uuid",
  "spent_seconds": 600,
  "answers": [{"question": "string", "answer": "string"}],
  "note_text": "string|null",
  "practice_result": "string|null",
  "current_datetime": "ISO datetime|null"
}

Response data:
{
  "completed_item": "RoadmapItem",
  "user_profile": "UserProfile",
  "xp_earned": {
    "pending_xp": 80,
    "status": "completed|pending_check",
    "global_xp": 0
  }
}


18. POST /api/roadmap/feedback
------------------------------
Logic:
- Saves user feedback.
- Sends profile, roadmap, selected items and feedback to LLM.
- LLM can update roadmap and replace 1-2 items.
- If LLM fails, feedback is still saved and response is 200.

Body:
{
  "telegram_id": 123,
  "roadmap_id": "uuid",
  "item_ids": ["uuid"],
  "feedback_type": "useful|not_suitable|too_hard|too_easy|already_completed|change_request",
  "feedback_text": "string|null",
  "max_items_to_change": 1,
  "dialog_history": [],
  "current_datetime": "ISO datetime|null"
}

Response data on LLM success:
{
  "saved_feedback": "RoadmapFeedback",
  "llm_output": {},
  "updated_roadmap": "Roadmap|null",
  "changed_items": ["RoadmapItem"],
  "pushes": ["MotivationPush"]
}

Response data when LLM failed:
{
  "saved_feedback": "RoadmapFeedback",
  "llm_output": null,
  "updated_roadmap": null,
  "changed_items": [],
  "pushes": [],
  "correction_status": "llm_failed",
  "correction_error": "string"
}


19. POST /api/notifications/send-due
------------------------------------
Logic:
- Gets due planned MotivationPush rows.
- Applies user notification settings.
- If dry_run=true, does not send Telegram messages.
- If dry_run=false, TELEGRAM_BOT_TOKEN is required.

Body:
{
  "telegram_id": 123,
  "limit": 20,
  "dry_run": true,
  "current_datetime": "ISO datetime|null"
}

Response data:
{
  "dry_run": true,
  "requested_telegram_id": 123,
  "processed": 0,
  "results": [
    {
      "push_id": "uuid",
      "telegram_id": 123,

      "reason": "push_disabled, only for cancelled",
      "message_text": "string, only for dry_run would_send",
      "button_text": "string|null, only for dry_run would_send",
      "sent_today": 3,

      "error": "string, only for failed",

Minimal frontend flow
=====================

1. POST /api/profile/analyze
Use user free text. Read:
- data.user_profile
- data.llm_output.Need_question OR data.fallback_output.Need_question
- data.llm_output.Next_question OR data.fallback_output.Next_question

2. If Need_question=true, ask the returned Next_question in UI.

3. PATCH /api/profile/{telegram_id}
Send button answers or manual edits.

4. POST /api/roadmap/generate
Read:
- data.created.roadmap
- data.created.items
- data.created.pushes

5. GET /api/profile/{telegram_id}/state
Use this as the main screen payload.

6. GET /api/roadmap/{roadmap_id}/item/{item_id}/test
Use this for the item completion/test screen.
