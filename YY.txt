Нужно использовать Aiogram 3.0
Пользователь переходит в телеграм бота. Пользователь хочет научится какомуто навыку (например Frontend разработчик, или дизайнер)
Он нажимает /start и ему высвечивается общая информация о боте, что бот делает и тд общая информация
У пользователя есть две кнопки: Составить трек, Информация
Информация. Пусть тут будет написана общая информация о том что будет делать бот, что он помогает обучатсья какомуто навыку и тд.
У пользователя есть кнопка "Составить трек"
После этого LLM должна задать вопросы:
    Шаг 1: Расскажи о своей цели изучения
    Шаг 2: Как бы ты оценил текущий свой уровень?
        (И тут и пользователя появляются 3 кнопки: Первая кнопка - Начинающий; Вторая кнопка - Есть базовые знания; Третья кнопка - Професионал)
    Шаг 3: Сколько времени в неделю ты готов уделять обучению?
        (И тут и пользователя появляются 3 кнопки: До 3 часов; До 3-7 часов; Более 7 часов)
    Шаг 4: Какой формат обучения тебе удобнее?
        (И тут и пользователя появляются несколько кнопок, он может выбрать несколько кнопок теость гибридно: Видео; Практические задания; Статьи)
    Шаг 5 (необязательный): бот спрашивает пожелания и ограничения по материалам
        Пример вопроса: "Есть ли пожелания или ограничения по материалам?"
        Кнопки:
        [Пропустить] [Больше практики] [Без длинных видео] [Больше коротких материалов] [Не показывать источник/автора] [Написать своё]
        Это сохраняется в USER_PROFILE:
            Wishes -> исходный текст пользователя
            Preference_json -> структурированные правила для LLM
    Важное утачнение: Если пользователь уже в пером вопрсое указал вопрос какие то шаги которые планировались в будущем, к примеру время на обучение, или какой у него уровень. В таких случая нам не нужно дальше отрисосовать некоторые вопрсы которые пользователь уже написал в 1 вопрсое.

    Вот примерный пример работы:

Когда всё собрано:
Prompt 2 создает маршрут напрямую на основе собранного профиля, истории диалога и интернет-поиска.
Prompt 2 учитывает Preference_json: запреты авторов/источников, лимит длинных видео, больше практики и другие пожелания.
Prompt 2 для каждого ресурса генерирует мини-проверку:
    Self_check_questions -> короткие вопросы для пользователя
    Completion_check_json -> тест/открытый вопрос/чеклист практики
    Min_seconds_before_complete -> минимальное время перед кнопкой "Пройдено"
    Xp_policy_json -> правила начисления XP
    Важно: полный XP не начисляется только за кнопку "Пройдено"; нужен мини-тест, конспект, практика или проект.
Prompt 2 возвращает JSON для сохранения в БД:
    user_profile_update -> USER_PROFILE
    roadmap -> ROADMAP
    roadmap_items -> ROADMAP_ITEM
    motivation_pushes -> MOTIVATION_PUSH


Пока создает трэк мы показываем какой то фалбек, допустим "Пожалуйста подождите"
Потом удаляется фалбек

Дальше приходит сообщение пользователю об успешном создании трека и кратко что у него будет
    пример:
        Твой план успешно на Frontend разработчика 
        Примерная длительность: 4 месяца
        Что будет в маршруте:
            • основы HTML и CSS
            • JavaScript
            • React
            • практические проекты
            • подготовка портфолио
    А ниже под этим сообщением будут кнопки: Скорекртировать, Создать занаво
    После этого сообщения НА КЛАВИАТУРЕ появляются две кнопки меин меню: Открыть карту, профиль, AI-мастер

После генерации маршрута бот также планирует мотивационные пуши.
Тон пушей: дерзкий, настойчивый, в стиле Duolingo, но без личных оскорблений и без сообщений ночью.
Примеры:
    "Твой streak смотрит на тебя с осуждением. Закрой один шаг сегодня."
    "XP сам себя не заработает. Один мини-тест — и ты снова в игре."
    "Материал ждет. Отмазки уже закончились."
Ограничения:
    максимум 3 пуша в день
    не отправлять в quiet hours
    если пользователь отключил пуши, не отправлять
    пуш должен вести к действию: открыть маршрут, пройти тест, закрыть дедлайн, сохранить streak

Дальше тут будет несколько сценаривем в зависимости от того что пользователь выберит (какие кнопки пользователь выбрал)
    Профиль.
         Пользователю юудет выводится:
            1) Персонаж 
            2) Стрик профиля 
            3) Навыки
            4) Прогресс по скилам
            5) Система достижения, Прогресс для XP и лидерборд
        Все эти кнопки это URL и они будут вести в мини эпп        
    Открыть карту.
        Просто URL
    Скорекртировать.
        Когда пользователь выбрит эту кнопку, его нужно српосить: Что бы ты хотел изменить в плане обучения?
            (Пользователь может ответтиь типо например: Удалить первый блок,я его знаю.)
    AI-мастер.
        Просто URL
